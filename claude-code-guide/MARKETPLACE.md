# Claude Code на максимум

> Справочник-настройка, чтобы выжимать из Claude Code максимум — конфиги, «Thinking»-режимы, отладка.

Оригинал из сообщества Claude Code Skills (zebbern), адаптирован и локализован командой COMANDOS.

## 😩 Знакомая боль?

- **«Поставил Claude Code, а он делает не то»** — каждый раз объясняешь одно и то же про свой проект, а агент всё равно лезет не туда.
- **«Агент застрял и крутится по кругу»** — повторяет одну и ту же ошибку, а как его вывести из ступора — непонятно.
- **«Ответы поверхностные»** — там, где нужно подумать, агент рубит с плеча и выдаёт первое, что пришло.
- **«Контекст забивается, агент тупеет»** — закинул весь проект, а он потерялся и стал промахиваться.
- **«Я не технарь, а тут какие-то конфиги»** — хочется просто, чтобы работало, а не разбираться в настройках по англоязычным докам.

Корень один: **Claude Code мощный, но без настройки работает вполноги.** Этот навык — твоя шпаргалка по тому, как его правильно настроить и о чём просить, чтобы получать результат с первого захода.

## 📘 Что делает этот навык

Сразу честно: это **не «сделай за меня», а «настрой и подскажи правильно»**. Навык не пишет за тебя код — он превращает агента в эксперта по самому Claude Code и подкладывает ему (и тебе) выжимку лучших практик:

- **Помогает собрать `CLAUDE.md`** — файл-правила проекта, который агент читает первым. Команды запуска, стиль кода, рабочий процесс — один раз настроил, дальше не повторяешься.
- **Даёт словарь «Thinking»-ключей** — фразы вроде «продумай шаг за шагом», «найди корневую причину», «спланируй до начала», которые включают у агента режим глубокого рассуждения вместо рубки с плеча.
- **Даёт чек-лист отладки** — что делать, когда агент застрял: очистить контекст, дать предельно конкретные пути и имена файлов, попросить проверить логи с подробным выводом.
- **Учит экономить контекст** — не вываливать весь код, а сначала находить нужные файлы через поиск. Меньше шума — меньше промахов.
- **Закрепляет лучшие практики** — мелкими шагами, с проверкой, с правкой ошибок «на лету» и записью усвоенного обратно в `CLAUDE.md`.

На выходе — **предсказуемый Claude Code**: настроенный под твой проект, реже промахивается, реже требует переделок.

## 🔄 Как это работает

| Блок | Что внутри | Зачем |
|------|------------|-------|
| **Конфиг `CLAUDE.md`** | Готовый шаблон: команды, стиль, workflow | Чтобы агент знал правила проекта и не переспрашивал |
| **«Thinking»-ключи** | Словарь фраз, включающих глубокое рассуждение | Чтобы в сложных местах агент думал, а не угадывал |
| **Отладка** | Чек-лист, когда агент застрял или делает не то | Чтобы быстро вернуть его в рабочее русло |
| **Практики** | Малые шаги, экономия контекста, фидбек-петля | Чтобы работа шла предсказуемо и без откатов |

> 💡 Главный приём — **итеративность с обратной связью**. Просишь маленькое изменение, проверяешь, поправляешь ошибку сразу и просишь агента «запомнить урок» в `CLAUDE.md`. Так агент с каждым разом работает точнее.

## 🎯 Когда включать

- Когда только осваиваешь Claude Code и хочешь сразу настроить его по уму.
- Когда заводишь новый проект и нужен грамотный `CLAUDE.md` с нуля.
- Когда агент застрял, повторяет ошибку или упорно делает не то.
- Когда ответы поверхностные и нужно включить режим глубокого мышления.
- Когда хочется перестать воевать с агентом и получать стабильный результат.

Не нужен, когда тебе надо что-то конкретное сделать в коде или контенте — для этого есть профильные навыки. Этот — про сам инструмент.

## 📦 Что на выходе

- Настроенный `CLAUDE.md` под твой проект — агент знает команды, стиль и рабочий процесс.
- Понимание, какими словами включать у агента глубокое рассуждение.
- Рабочий чек-лист, как вытаскивать агента из ступора.
- Привычка работать итеративно и экономить контекст — меньше промахов, меньше переделок.
